Service Booking System

The Essence of Object-Oriented Design (OOD):

The Object-Oriented Design (OOD) principles play a pivotal role in the Service Booking System, shaping its architecture and functionality.

User-Centric Approach with Classes:

The project adopts a user-centric approach by organizing functionalities into classes. Each menu item on the navigation bar corresponds to a specific class, encapsulating the functionality associated with that particular aspect of the system.

SignUp - Encapsulation and Abstraction:

The signUp() method in the User class embodies the encapsulation and abstraction principles of OOD. It encapsulates the logic for user registration, providing a clear separation of concerns and maintaining a clean interface for user interactions.

Service Discovery - Polymorphism:

The SearchServices class showcases the polymorphic nature of classes. It adapts to various search queries, triggering method calls that filter through available services, providing a versatile and dynamic user experience.

Booking Services - Dynamic Relationships:

Booking a service involves interactions between the Customer and ServiceProvider classes. The Booking class orchestrates the acceptance or rejection process, symbolizing dynamic relationships between classes and showcasing the elegance of OOD.

Rating - Encapsulation:

The ReviewService class encapsulates the logic for reviewing and rating services. User feedback is neatly stored within instances of the ServiceProvider class, demonstrating the encapsulation principle of OOD.

Admin - Hierarchy and Control:

The Admin class, serving as the project administrator, manages the system with a clear hierarchical structure. From auto-approving services to disabling them, the Admin class embodies control and governance over the system.
